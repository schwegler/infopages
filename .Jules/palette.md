## 2026-04-02 - Avoid Tabindex on Static Text Elements
**Learning:** Adding `tabindex="0"` and focus styles to static, non-interactive `<div>` or text elements is an accessibility anti-pattern. While it might visually seem like an enhancement to show focus rings, it violates WCAG guidelines because keyboard and screen reader users expect focusable elements to be actionable. This creates unnecessary "dead stops" during navigation and disrupts the flow.
**Action:** When adding focus states (`:focus-visible`), only apply them to natively interactive elements like links (`<a>`), buttons (`<button>`), form inputs, or complex custom widgets that act like them and manage their own focus. Remove any manually added `tabindex` on purely informational cards.
